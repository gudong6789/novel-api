# 小说管理后台 PC 端 - 使用指南

## 📥 安装与运行

### 方式一：使用打包好的安装包

1. **下载安装包**
   - 从项目发布页面下载 `小说管理后台 Setup x.x.x.exe`

2. **安装程序**
   - 双击运行安装包
   - 选择安装路径
   - 点击"安装"

3. **启动应用**
   - 桌面双击快捷方式
   - 或从开始菜单启动

---

### 方式二：从源码运行 (开发模式)

**前提条件:**
- 安装 Node.js 18+ : https://nodejs.org/

**步骤:**

1. **打开命令行**

## 🎯 功能使用

### 1. 登录系统

1. 打开应用
2. 输入管理员账号
   - 用户名：`admin`
   - 密码：部署时设置的密码
3. 勾选"记住登录状态" (可选)
4. 点击"登录"

---

### 2. 数据概览

登录后默认显示数据概览页面，包含：

- **授权码总数** - 已生成的所有授权码数量
- **已激活** - 已被设备使用的授权码
- **设备总数** - 注册的设备总数
- **在线设备** - 当前在线的设备数
- **今日阅读** - 所有设备今日累计阅读时长
- **累计阅读** - 历史总阅读时长

**快捷操作:**
- ➕ 生成授权码
- 📦 批量生成
- 🔄 刷新数据

---

### 3. 授权码管理

#### 3.1 单个生成

1. 点击"授权码管理"
2. 点击"➕ 生成授权码"
3. 选择有效期:
   - 1 天
   - 7 天
   - 30 天
   - 永久
4. 填写备注 (可选)
5. 点击"生成"
6. 复制生成的授权码

#### 3.2 批量生成

1. 点击"📦 批量生成"
2. 输入生成数量 (1-1000)
3. 选择有效期
4. 填写备注 (可选)
5. 点击"批量生成"
6. 复制所有授权码 (可一键复制到剪贴板)

#### 3.3 封禁/解封

1. 在授权码列表中找到目标授权码
2. 点击"封禁"按钮 (红色)
   - 已封禁的授权码无法使用
3. 点击"解封"按钮 (绿色)
   - 恢复授权码的使用权限

#### 3.4 删除授权码

1. 点击"删除"按钮
2. 确认删除
3. 授权码被永久删除 (不可恢复)

---

### 4. 设备管理

查看已注册设备的信息：

- **设备名称** - 用户设置的设备名称
- **型号** - 设备型号
- **状态** - 🟢 在线 / ⚪ 离线
- **授权码** - 使用的授权码
- **最后活跃** - 最后一次上报时间
- **今日阅读** - 今日累计阅读时长
- **累计阅读** - 历史总阅读时长

---

### 5. 全局配置

远程修改阅读客户端的运行参数：

#### 时间设置
- **启动时间 - 开始**: 每日自动启动的开始时间 (小时)
- **启动时间 - 结束**: 每日自动启动的结束时间 (小时)

#### 滑动设置
- **滑动间隔最小值**: 两次滑动之间的最小间隔 (毫秒)
- **滑动间隔最大值**: 两次滑动之间的最大间隔 (毫秒)

#### 阅读时长
- **单次阅读最小**: 每次阅读的最小时长 (毫秒)
- **单次阅读最大**: 每次阅读的最大时长 (毫秒)

#### 时长限制
- **单 APP 单日最大**: 单个 APP 每日最大阅读时长 (毫秒)
- **单日总时长**: 所有 APP 每日总时长限制 (毫秒)

**保存配置:**
1. 修改参数
2. 点击"💾 保存配置"
3. 配置立即生效 (已在线设备下次请求时更新)

---

### 6. 系统设置

#### API 配置
- **API 基础地址**: 云端 API 服务地址
- 默认：`https://novel-api-production-55af.up.railway.app/api`
- 修改后点击"保存 API 配置"

#### 关于
- 软件名称
- 版本号
- 构建时间
- 技术栈信息

## 💡 使用技巧

### 快捷操作

1. **快速刷新数据**
   - 在任意页面按 F5 刷新

2. **批量导出授权码**
   - 批量生成后点击"复制所有授权码"
   - 粘贴到 Excel 或文本文件

3. **查看设备详情**
   - 设备列表中可以看到详细的阅读数据

4. **快速定位问题设备**
   - 查看"最后活跃"时间
   - 离线时间过长可能设备有问题

## 🐛 故障排查

### 问题 1: 无法登录

**可能原因:**
- API 地址错误
- 网络连接问题
- 账号密码错误

**解决方法:**
1. 检查 API 地址配置
2. 检查网络连接
3. 确认账号密码正确

---

### 问题 2: 数据加载失败

**可能原因:**
- API 服务不可用
- Token 过期

**解决方法:**
1. 退出重新登录
2. 检查 API 服务状态
3. 刷新页面

---

### 问题 3: 应用无法启动

**可能原因:**
- Node.js 未安装
- 依赖未安装

**解决方法:**
```bash
# 重新安装依赖
npm install

# 重新启动
npm start
```
